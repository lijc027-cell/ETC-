# V1 查询流程与回答总结

本文总结 `etf-query` 当前 v1 的整体查询链路，以及最终回答是怎么生成的。

## 一眼看懂

```text
用户问题
  -> 意图识别（含复合意图拆解）
  -> 提取 fundcode / period（单只查询）或解析筛选/搜索条件（多只查询）
  -> 没有 fundcode 时先做中文名称反查
  -> 读取 schema.md，生成 pymongo 查询代码
  -> SSH 执行远端 Mongo 查询（可跨集合）
  -> 格式化结果
  -> 输出结构化回答
```

## 1. v1 的定位

v1 做一件事：**把用户自然语言稳稳翻译成安全的 pymongo 查询，再由本地规则把数据库结果压成简短结构化回答（Text2MongoDB）**。

它支持：

- 基本信息、基金类型/投资类型
- 基金规模、净值
- 跟踪指数、业绩比较基准
- 收益率（全周期）及同类/ETF 排名
- 费率（管理费率 + 托管费率）
- 基金经理（现任详情：任职起始日、年化回报、任职天数、管理规模）
- 分红（累计总额 + 次数）
- 申购赎回状态、联接基金
- 投资目标/范围/理念/策略/风险特征
- 成立日期
- **搜索**（按名称/指数名模糊匹配，返回候选列表）
- **条件筛选**（按规模/费率/收益率/上市地点/成立日等多条件过滤+排序）
- **多只对比**（按用户指定维度并排展示）
- **持仓/报告展开**：
  - 季报：前N大行业、重仓概念
  - 年报：前十大重仓股/证券、年报行业+占比、机构持仓比例与份额、投资风格、净资产变动

它不支持：

- 实时行情（成交额、融资余额等交易类指标不在查询计划范围内）
- 个股分析（引导至对应 skill）

## 2. 数据来源

v1 查询覆盖三个 MongoDB 集合：

| 集合                          | 内容                                                                                   |
| --------------------------- | ------------------------------------------------------------------------------------ |
| `tb_ths_etf_base`           | 基础信息、规模、净值、收益率及排名（含数字型和字符串型排名）、费率、基金经理（含独立子字段）、分红、交易类指标（成交额/净现金流/融资余额/融券卖出）、同类平均收益率等 |
| `tb_ths_etf_report_quarter` | 季报：前N大行业（`ths_top_n_top_industry_name_fund`）、重仓概念（`ths_zcgnmc_fund`）                 |
| `tb_ths_etf_report_year`    | 年报：重仓股/证券代码、年报行业+占比、机构持仓、投资风格、净资产变动                                                  |

季报/年报字段均为 array，元素格式 `{value: ..., rank_num: 排名}`，取最新一期时按 `year_num` + `type_num` 降序取第一条。

## 3. 总体流程

```text
用户问题
  -> 意图识别（单意图 or 复合意图拆解）
  -> 必要时做中文名称反查（单只查询无代码时）
  -> 读取 schema.md（集合结构 + few-shot，每次必读）
  -> 生成 pymongo 查询代码（find / find_one / $in / $regex / sort / limit）
  -> 读取 executor.md，SSH 远端执行
  -> 结果格式化
  -> 输出最终回答
```

## 4. 逐步流程

### 4.1 意图识别

v1 先做意图分类，确定：

- **目标集合**：`tb_ths_etf_base` / `tb_ths_etf_report_quarter` / `tb_ths_etf_report_year`（可跨集合）
- **查询模式**：单只标量查询 / 搜索 / 条件筛选 / 多只对比
- **复合意图**：一句话含多个意图时，全部拆解，合并字段单次查询（同集合）或分集合分别查询后合并输出

常见意图分类见下表（完整列表见 `skill.md` 意图识别节）：

| 意图类型   | 示例触发词                                            | 查询模式                     |
| ------ | ------------------------------------------------ | ------------------------ |
| 基本信息   | `是什么`、`介绍一下`、ETF代码单独出现                           | find_one                 |
| 收益率+排名 | `近1周/1月/3月/6月/1年/2年/3年/5年`、`今年以来`、`成立以来`、`表现怎么样` | find_one + 取净值时间序列计算时间范围 |
| 费率     | `管理费`、`托管费`、`费率贵吗`                               | find_one                 |
| 基金经理   | `谁在管`、`任职回报`、`换经理了吗`                             | find_one                 |
| 净值日期   | `最新净值哪天`、`数据更新到什么时候`                             | find_one 取净值序列最后一条 btime |
| 交易指标   | `成交额`、`净现金流`、`融资余额`、`融券卖出`                       | find_one                 |
| 同类平均   | `同类平均收益`、`同类均值`                                  | find_one                 |
| 基金份额   | `份额多少`、`总份额`                                     | find_one                 |
| 持仓/季报  | `行业配置`、`重仓概念`、`题材`                               | find_one（季报集合，取最新一期）     |
| 持仓/年报  | `重仓股`、`前十大`、`机构持仓`、`净资产变动`、`投资风格`                | find_one（年报集合，取最新一期）     |
| 搜索     | `帮我找`、`XXX赛道ETF`、`有哪些`                           | find + $regex，limit 20   |
| 条件筛选   | `规模最大`、`费率最低`、`上交所ETF排行`                         | find + sort + limit      |
| 多只对比   | `对比`、`vs`、`哪个更好`                                 | find + $in               |

### 4.2 中文名称反查

仅在**单只查询且无 6 位数字代码**时触发。

- 把中文名解析成 `fundcode`
- 匹配多只时返回候选列表，由用户确认
- 完全无匹配时报错

### 4.3 读取 schema.md（每次必读）

每次执行前读取 `references/schema.md`，获取：

- 集合结构与字段类型
- 样例数据（防止字段名拼写错误）
- few-shot 查询示例

**不得凭记忆构造查询。**

### 4.4 生成 pymongo 查询代码

根据意图和集合，生成对应 pymongo 代码：

| 场景      | 主要操作                                                                                                         |
| ------- | ------------------------------------------------------------------------------------------------------------ |
| 单只标量查询  | `find_one({"fundcode": "510300"}, {"projection": {...}})`                                                    |
| 搜索（模糊）  | `find({"ths_name_of_tracking_index_fund": {"$regex": "科技"}}, ...).sort("ths_fund_scale_fund", -1).limit(20)` |
| 条件筛选    | `find({"ths_manage_fee_rate_fund": {"$lt": 0.15}}).sort(...).limit(20)`                                      |
| 多只对比    | `find({"fundcode": {"$in": ["510300", "510500"]}}, ...)`                                                     |
| 季报/年报   | `find_one({"fundcode": "510300"}).sort("year_num", -1).limit(1)`                                             |
| 跨集合复合意图 | 分别对 `tb_ths_etf_base` 和报告集合各执行一次查询，结果合并输出                                                                    |

### 4.5 SSH 远端 Mongo 查询

读取 `references/executor.md`，通过 SSH 将查询代码传至远端，远端 Python runner 直连 MongoDB 执行只读查询，结果取回本地。

### 4.6 结果格式化

格式化规则：

- 规模/市值/净资产变动/经理管理规模：÷ 1e8，保留2位小数，加"亿"
- 收益率/费率/占比/净资产变动率：保留2位小数，加"%"
- 排名字段：优先展示 `_fund_origin`（如"100/500"），不拼接数字字段
- **时间范围计算**：查询 `ths_unit_nv_fund` 时间序列（列表倒序，最新在最后）
  - 最新一条 `btime` = 截止日，往前数对应交易日数的 `btime` = 起始日
  - 映射：近1周=5条、近1月=21条、近3月=63条、近6月=126条、近1年=250条、近2年=500条、近3年=750条、近5年=1250条
  - 今年以来=取当年第一条，成立以来=取第一条
- 时间序列/报告集合：取最新一期（`year_num` + `type_num` 最大）
- 字段为 null：标注"暂无数据"，不编造
- 年报重仓股/证券：只有代码，直接展示，不捏造名称

如果没查到数据：

```text
未在 ETF 数据库中找到代码 XXXX 对应的 ETF。
```

## 5. 最终回答格式

v1 的最终回答是结构化模板输出，不是 LLM 自由生成的长文。

| 意图    | 输出格式                                                                                       |
| ----- | ------------------------------------------------------------------------------------------ |
| 基本信息  | 标题行（名称+代码）+ 关键字段列表                                                                         |
| 收益率   | 表格：周期 \| 收益率 \| 同类排名 \| ETF排名 \| 时间范围（起始日期 ~ 截止日期，从 `ths_unit_nv_fund` 时间序列取对应索引的 `btime`） |
| 持仓    | 表格：排名 \| 行业名/股票代码 \| 占比%                                                                   |
| 多只对比  | 横向表格：行为指标，列为基金                                                                             |
| 复合意图  | 分段展示，每段加小标题                                                                                |
| 搜索/筛选 | 列表：代码 \| 名称 \| 规模 \| 费率（\| 相关排序字段）                                                         |

## 6. 歧义消解（不向用户反复确认）

| 歧义场景              | 默认处理                                                   |
| ----------------- | ------------------------------------------------------ |
| "持仓了哪些行业"未指定季报/年报 | 默认季报，取最新一期                                             |
| "重仓了什么"未区分股票/行业   | 默认年报重仓股                                                |
| "前十大"未说股票还是行业     | 默认年报前十大重仓股                                             |
| "规模"未区分基金规模/总市值   | 默认 `ths_fund_scale_fund`，有 `ths_current_mv_fund` 时同时展示 |
| 未指定收益率周期          | 返回全周期                                                  |
| 未指定报告期            | 取最新一期                                                  |
| 搜索未指定返回数量         | limit 20，按规模降序                                         |

## 7. 运行结果结构

`semantic_query()` 最终返回结构化 payload，常见字段：

- `question`：原始问题
- `entities`：抽取的实体（fundcode、period、筛选条件等）
- `intent`：识别到的意图（含复合意图列表）
- `collections`：本次查询涉及的集合
- `pymongo_code`：生成的查询代码
- `result`：Mongo 原始返回
- `answer`：格式化后给用户看的最终回答
- `debug`：调试链路

## 8. 一句话总结

v1 的核心不是"让模型回答 ETF 问题"，而是：

> 先把自然语言稳稳翻译成安全的 pymongo 查询，再由本地规则把数据库结果压成简短结构化回答。
